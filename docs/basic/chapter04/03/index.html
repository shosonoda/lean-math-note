    Set: 型の上の集合¶

    Lean は型理論を基礎にしているため，数学でいう「対象の集まり」はまず型として現れます． たとえば自然数全体は Nat，実数全体は ℝ という型です． 一方，Mathlib の Set α は「型 α の部分集合」を表します． 定義は非常に単純で，Set α は α → Prop として実装されています．

    ```lean4:Mathlib.Data.Set.Defs def Set (α : Type u) := α → Prop 

    つまり，`s : Set α` とは，各 `x : α` に対して「`x` が `s` に属する」という命題を返す述語です．
所属 `x ∈ s` は，内部的には `s x` と同じ意味です．
この意味で，Lean の `Set α` は数学の「集合」そのものというより，固定された型 `α` 上の部分集合全体，つまり冪集合 `$\mathcal P(α)$` に対応します．
ただし実際の証明では，`Set` が関数として実装されていることを直接展開するより，`{x | p x}` や `x ∈ s` や `s ⊆ t` のような集合のインターフェースを使うのが普通です．

対応を大まかに書くと，次のようになります．

* 数学の全体集合・台集合: Lean では多くの場合 `α : Type*`
* 数学の部分集合 `s \subseteq α`: Lean では `s : Set α`
* 数学の冪集合 `$\mathcal P(α)$`: Lean では `Set α` という型
* 条件 `x \in s`: Lean では命題 `x ∈ s`

このため，`Set Nat` は「自然数という集合」ではなく，「自然数の部分集合を要素にもつ型」と読む方が正確です．
Mathlib のドキュメントでは `Set X` と呼ばれますが，直感的には `Subset X` と呼んだ方が意味が分かりやすい場面もあります．

```lean4
#check Set
#check Set.powerset
#check Set.ext
#check Set.mem_setOf
#check Set.setOf_bijective

def smallSet : Set Nat :=
  {n | n < 3}

example (n : Nat) : n ∈ smallSet ↔ n < 3 := by
  rfl

example : 2 ∈ smallSet := by
  norm_num [smallSet]

example : 4 ∉ smallSet := by
  norm_num [smallSet]

example (s t : Set Nat) : t ∈ Set.powerset s ↔ t ⊆ s := by
  exact Set.mem_powerset_iff t s

    集合の等式は，通常，外延性で証明します． 数学で「任意の x について x ∈ A と x ∈ B が同値だから A = B」と言う議論に対応するのが Set.ext や tactic の ext です．

    example (A B : Set Nat) : A ∩ B = B ∩ A := by
  ext x
  simp [and_comm]

    Set α と似て見えるものに Subtype があります． s : Set α は α 上の述語であり，それ自体は Set α 型の項です． 一方，{x : α // x ∈ s} は，s に属する元だけを集めた新しい型です． 要素は値 x : α と証明 x ∈ s の組です．

    Mathlib では s : Set α を型として使うと，この subtype へ coercion されます． たとえば x : smallSubtype は「smallSet の元」としてのデータであり，(x : Nat) に戻すと元の自然数が得られます．

    abbrev smallSubtype : Type :=
  {n : Nat // n ∈ smallSet}

#check Subtype
#check Set.coe_eq_subtype
#check Subtype.mem

example : (⟨2, by norm_num [smallSet]⟩ : smallSubtype).1 = 2 := by
  rfl

example (x : smallSubtype) : (x : Nat) ∈ smallSet := by
  exact x.property

    数学では Nat や ℝ を「集合」と呼ぶことが多いですが，Lean の基礎ではそれらは型です． これは単なる用語の違いではありません． 型理論では，型を ZFC の集合として素朴に解釈すると扱いにくい型も自然に現れます． たとえば関数型や高階の型を含む定義では，ある型 P と，P から作った型 T P が同型になるような自己参照的な状況を考えられます． 擬似コードで書くと，次のような例です．

    data Fix f = Fix (f (Fix f))
data T a = T ((a -> Bool) -> Bool)
type P = Fix T

    これは Lean の帰納型としてそのまま導入する例ではありませんが，型を集合として読むことの限界を示す標準的な警告として有用です． もしそれらを素朴な集合と同一視し，濃度だけで

    |P| = |T P| = 2^(2^|P|)

    のように読んでしまうと，Cantor 型の議論と衝突します． Lean で日常的に数学を形式化するときにこの病理を直接扱うことは多くありませんが， 「Lean の Type は単に大きな集合論的宇宙の部分集合だ」と考えるのではなく， 「対象が住む型」と「その型の上の部分集合 Set α」を分けて読むことが重要です．

    以後の章で位相空間の開集合，閉集合，コンパクト集合，零点集合などを書くときは，ほとんどの場合 Set X を使います． 代数の部分群や部分加群は，単なる Set ではなく，Set に閉性条件を束ねた構造体として現れます．
